Lorebook Merger
Combine two lorebooks into one. The primary lorebook's settings are kept; the secondary's entries are appended to the end.
How to use
  1. Export both lorebooks from DreamJourney and paste each into a box below.
  2. The primary lorebook keeps its name and top-level settings.
  3. The secondary lorebook's entries are added after the primary's.
  4. Click Merge, then Copy output and import it back into DreamJourney as a new lorebook.
  5. Tip: check for duplicate entry names afterwards, as merging does not deduplicate.

Wrap All Triggers (Cascade Destroyer)
Finds every trigger in your lorebook and wraps each one wherever it appears inside entry bodies. This stops entries from triggering each other (cascade) while keeping the text readable to the model.
How to use
  1. Paste your full lorebook JSON.
  2. Any entry you want to keep cascading (e.g. a deliberate hub entry) goes in the exempt box, one name per line, matched exactly.
  3. Click Wrap all triggers. Every trigger word found in non-exempt bodies gets wrapped like _word_.
  4. Copy the output and import it back into DreamJourney. The "References" list on each entry should now be empty (except exempt entries).

Wrap a Snippet (Cascade Buster)
For fixing one entry at a time. Extracts your lorebook's triggers, then wraps any of those triggers found in a block of text you paste, without touching the rest of the lorebook.
How to use
  1. Paste your lorebook JSON and click Extract triggers to build the trigger list (review it in the audit box).
  2. Paste the body text you want to clean into Text to process, usually a single entry's description.
  3. Click Wrap triggers. Matching triggers get wrapped like _word_ (spaces inside multi-word triggers become underscores).
  4. Copy the result and paste it back into that one entry on DreamJourney.

Remove Wrapping (Recascadanator)
The undo button. Strips all underscore wrapping from entry descriptions, restoring normal text and re-enabling cascade. Useful if you wrapped a lorebook and want to start over.
How to use
  1. Paste your wrapped lorebook JSON.
  2. Click Remove wrapping. An underscore between two characters (like car_to_work) becomes a space; all other underscores are removed.
  3. Only entry descriptions are touched. Triggers, weights, pins and everything else are left exactly as-is.
  4. Copy the output and import it back into DreamJourney.

Format Checker
Not importing into DreamJourney? Paste your lorebook here to check it against the exact format DreamJourney requires. This finds the small mistakes (missing fields, an entry with no triggers, a stray comma) that quietly break an import.
How to use
  1. Paste your full lorebook JSON below.
  2. Click Check format.
  3. Green means it's good to import. Red lists every problem found, with the entry name so you know where to look.
  4. Fix the issues in your lorebook, then check again until it passes.
What a valid lorebook needs
  1. A top level with name, thumbnail, ispublic and an entries list.
  2. Every entry needs name, description, type, weight, pinned, hidden and a keys list.
  3. Every entry must have at least one trigger (a key with keyText). Entries can be any length.
